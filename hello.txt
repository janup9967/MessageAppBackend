📘 Use Case: WhatsApp-like Real-Time Messaging App

🎯 Objective
Build a secure, scalable messaging platform with real-time communication, multi-device sync, and contact-based conversation management.


✅ Functional Requirements

🔧 Backend (ASP.NET Web API + SQL Server + SignalR + Swagger)

1. Authentication
- Register/Login using JWT
- Role-based access (Admin/User)
- Refresh token support
- Email verification, forgot/reset password

2. User Management
- Get user profile
- Update profile
- Online/offline status tracking

4. Contact Management
Add contact (user-to-user relationship)
Fetch all contacts for logged-in user
Remove contact (optional)

3. Messaging
- Send message (text/media)
- Fetch conversation history
- Mark message as read
- Delete/edit message (optional)
- Store messages using SQL Server stored procedures

4. Real-Time Communication
- SignalR Hub for:
  - New message delivery
  - Read receipts
  - Typing indicators
  - Online status updates
  - Join/leave conversation groups based on selected contact

5. Media Handling
- Upload images/videos/documents
- Store in file system or cloud (e.g., Azure Blob Storage)
- Link media to messages in DB

6. Swagger Integration
- Document all API endpoints
- Enable testing via Swagger UI

🎨 Frontend (React or HTML/JS)

1. Authentication UI
- Login/Register
- Email verification
- Password reset

2. Chat Interface
- List of conversations
- Real-time message updates
- Read/unread indicators
- Typing status
- Media previews

3. SignalR Integration
- Connect to SignalR Hub
- Emit and listen for:
  - New messages
  - Read receipts
  - Typing events

4. Responsive Design
- Mobile-first layout
- Adaptive chat UI

5. Contact List UI
- Display all contacts
- Click to open conversation with selected contact

🧱 Architecture Overview

Client (React/HTML) <--> ASP.NET Web API <--> SQL Server (Stored Procedures)
                            |
                         SignalR Hub
                            |
                   Real-Time Sync Across Clients

🛡️ Non-Functional Requirements

- Secure endpoints with JWT
- Scalable backend (SignalR + SQL Server)
- Efficient media handling
- CORS setup for frontend-backend communication
- Logging and error handling

🧪 Testing Strategy

- Swagger for API testing
- Unit tests for stored procedures
- Manual testing in multiple browsers
- Load testing for SignalR connections
- UI testing for contact selection and chat flow
